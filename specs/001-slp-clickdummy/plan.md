# Implementation Plan: SLP Clickdummy — Elektrische Lastgang-Verwaltung

**Branch**: `001-slp-clickdummy` | **Date**: 2026-03-21 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `specs/001-slp-clickdummy/spec.md`

## Summary

A locally-executable, browser-based MVP application (Python + Streamlit) for creating,
managing, combining, visualising and exporting annual electrical load curves (Lastgänge).
The application supports multiple projects with mandatory metadata and full JSON
round-trip persistence. Load series are generated from five source types: BDEW Standard
Load Profiles (SLP), measured CSV/XLSX uploads, hierarchical Power Summary models (PS),
simplified PV generation profiles (PVA), and BESS storage simulations. All series are
normalised to a 15-minute UTC time grid for the target year (current year − 2), combined
into an interactive Plotly chart with static limit lines, and exported as wide-format
CSV or project JSON.

## Technical Context

**Language/Version**: Python 3.11 / 3.12  
**Primary Dependencies**: Streamlit (UI + session state), pandas (time series), numpy (simulation), plotly (interactive chart), openpyxl (XLSX input, optional)  
**Storage**: File-based only — JSON download/upload for project persistence; CSV download for time series export; SQLite only if session-persistent state storage is needed beyond Streamlit session_state  
**Testing**: pytest + pytest-cov (unit/integration); hypothesis for property-based tests on numeric generators; pytest-benchmark for PERF-* benchmarks  
**Target Platform**: Local machine, any OS; launched via `streamlit run app.py`; single-user, no network required  
**Project Type**: Local web-app (Streamlit single-page application)  
**Performance Goals**: Chart update ≤ 1 s for ≤ 10 series × 35 040 steps; CSV parse ≤ 5 s for 10 MB file; app startup ≤ 3 s  
**Constraints**: No external API calls; fully offline; browser memory ≤ 500 MB for 10 series; deterministic outputs via project seed  
**Scale/Scope**: Single-user sessions; up to ~20 load series per project; 35 040 time steps per series; target year = current year − 2

## Complexity Tracking

| Violation | Why Needed | Simpler Alternative Rejected Because |
|-----------|------------|--------------------------------------|
| WCAG 2.1 AA (UX-003) cannot be fully met in Streamlit | Streamlit widgets have limited native accessibility attributes; focus management for dynamic forms is partially handled by the browser | Replacing Streamlit with a full frontend framework is out of scope for MVP; partial compliance is documented and the gap is limited to focus-order edge cases, not blocking interactions |
| `normalizer.py` contains >40-line mapping logic for ISO-week/KW53 edge cases | The KW53→KW52 mapping, leap-day removal, and "last-wins" collision resolution cannot be decomposed further without losing the temporal context needed for correctness | Splitting into sub-functions would require passing large state objects; the block is fully documented with inline rationale per rule |

## Phase 0 — Research (completed inline)

Key decisions validated during research (see `research.md` for full detail):

1. **BDEW SLP algorithm**: Normalised profiles scaled by `annualEnergy / profile_sum`. Saison
   boundaries and Tagesart (Werktag/Samstag/Sonntag) are pre-computed for the target year.
   Bundled as static CSV files — no external data fetch needed.

2. **Time grid normalisation**: Target year = `current_year − 2` (ensures full BDEW profile
   availability). Grid = DatetimeIndex at 15 min, UTC, Jan 1 00:00 → Dec 31 23:45 (35040
   steps; leap years retain 35136 steps unless leap-day removal is applied per spec).
   ISO-week 53 maps to week 52 (last week of year). Collisions: last series wins. Missing: 0.

3. **PV generator**: Synthetic profile using declination/hour-angle model seeded by
   `project.seed`. Azimuth + tilt encoded as cosine correction. Annual yield validated in
   [900, 1100] kWh/kWp/a range for central Europe defaults.

4. **BESS simulator**: Discrete time-step simulation (forward pass over 35040 steps).
   State-of-charge tracked per step. Strategy dispatch functions are separate pure functions
   (Peak Shaving: clip net > threshold; Eigenverbrauch: charge on PV surplus; Arbitrage:
   charge in low-price slots — price curve synthetic, seed-driven for MVP).

5. **Streamlit session state**: `st.session_state` holds a dict of `{uuid: Project}`. JSON
   import/export use `st.download_button` / `st.file_uploader`. No SQLite needed for MVP
   (single-session, in-memory only); SQLite path reserved for future multi-session support.

6. **Plotly chart**: `go.Scattergl` (WebGL) for performance at 35040 × 10 series.
   `uirevision` preserves zoom/pan state across Streamlit reruns.
